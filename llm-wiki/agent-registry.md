# agent-registry — 에이전트 역할 정의

> 갱신: 2026-05-16

## 에이전트 역할 분리 원칙

이 프로젝트는 **Claude(구현)** 와 **Codex(검증)** 두 AI를 분리 사용한다.
같은 에이전트가 구현과 검증을 동시에 하면 자기 실수를 못 잡는다.

---

## Claude (구현 담당)

| 항목 | 내용 |
|------|------|
| 역할 | 모듈 구현, 타입 작성, 설정 파일, llm-wiki 문서 |
| 금지 | 자기가 만든 코드를 자기가 최종 검증하는 것 |
| 진입점 | Claude Code CLI (`claude` 명령어) |

### Claude가 할 수 있는 것
- `src/` 모듈 코드 작성
- `src/types/` 스키마 정의
- `llm-wiki/` 문서 작성 및 갱신
- `prompts/` 프롬프트 템플릿 작성
- git commit & push

---

## Codex (검증 담당)

| 항목 | 내용 |
|------|------|
| 역할 | 구현된 코드 리뷰, 테스트 작성, 타입 안전성 검증 |
| 금지 | 구현 코드 직접 수정 (리뷰만 가능) |
| 진입점 | `mcp__codex__codex` 툴 |

### Codex에게 넘기는 기준
- 모듈 구현이 완료된 직후
- 타입 체크는 통과했지만 로직 버그 가능성이 있을 때
- 프롬프트 템플릿이 실제로 올바른 JSON을 반환하는지 확인할 때

### Codex 호출 형식 예시
```
검증 요청: src/intake/index.ts
- parseInquiry()가 빈 입력에서 안전하게 동작하는지 확인
- extractField() 정규식이 한국어 레이블을 정확히 인식하는지 확인
```

## 사람 (승인 담당)

| 항목 | 내용 |
|------|------|
| 역할 | 고객 발송 메시지 최종 승인, 견적 확정 |
| 금지 | 승인 없이 메시지 발송 |
| 인터페이스 | `src/approval/` 모듈 (Phase 3 예정) |

고객에게 보내는 모든 메시지는 `ApprovalRecord.state === 'approved'` 상태여야만 발송된다.
